# Reproducing the SaFE-Scale results

End-to-end recipe for going from the raw RadSaFE-200 dataset to the
headline numbers in the paper using only the files shipped in this release.

> Prerequisites:
> - `$WORKSPACE` set to the repository root.
> - The two apptainer images in `environment/` built
>   (`rag.sif`, `inference.sif`).
> - The RadSaFE-200 dataset staged as
>   `$WORKSPACE/datasets/risk_radiorag_unified.jsonl`.
> - A local HuggingFace model cache at `$MODELS_DIR` containing the
>   open-weight models listed in `phase3_inference/config.py`.
> - Optional: `HTTP_PROXY` if your cluster requires an outbound proxy,
>   and `HF_TOKEN` for gated HuggingFace repos.

## Pipeline overview

```
datasets/risk_radiorag_unified.jsonl
        │
        ▼   Phase 2 — RAG context construction
phase2_rag_context/final_datasets/risk_radiorag_final.jsonl
        │   (packs zero_shot + top_1/5/10 + padded_top_1 + evidence_* + extended_150k)
        │   (Phase 3 slices extended_150k into context_32k / context_100k / context_max)
        ▼   Phase 3 — Multi-condition inference (open-weight models)
phase3_inference/results/<model>/risk_radiorag.jsonl
        │
        ▼   Phase 4 — LLM-judge answer verification (Mistral-Small judge)
phase4_checking_results/results/<model>/risk_radiorag_checked.jsonl
        │
        ▼   Phase 5 — Bootstrap statistical analysis
phase5_bootstrapping/bootstrap_results/*.json   → Tables 1–4 in the paper
```

## Phase 2 — RAG context construction

One-shot pipeline from the Radiopaedia dump to a single enriched JSONL
carrying all context conditions per question.

```bash
SIF=$WORKSPACE/environment/rag.sif
EXEC="apptainer exec --bind $WORKSPACE:$WORKSPACE $SIF python3"

$EXEC phase2_rag_context/scripts/01_chunk_radiopaedia.py
apptainer exec --nv --bind $WORKSPACE:$WORKSPACE $SIF python3 \
    phase2_rag_context/scripts/02_build_embeddings.py
$EXEC phase2_rag_context/scripts/03_build_faiss_index.py
$EXEC phase2_rag_context/scripts/04_retrieve_context.py
$EXEC phase2_rag_context/scripts/05_add_context_to_datasets.py
```

The only GPU-requiring step is `02_build_embeddings.py` (BGE-large on
~150 k Radiopaedia chunks, ~30 min on a single A100).

## Phase 3 — Inference

Model list (`MODELS` in `phase3_inference/config.py`) ships only the
**locally-hostable open-weight models** from the paper — the models that
fit on a single 8×GPU node served by vLLM. The full leaderboard sweep for
these models is one SLURM job array:

```bash
sbatch phase3_inference/slurm/run_inference_array.sh
```

Each array task:

1. Reads its model config by index from `config.py`.
2. Starts a fresh vLLM server against that model (`--bind $MODELS_DIR:/models`).
3. Runs `02_run_inference.py`, which evaluates the model on every
   applicable condition × (greedy + 20-sample stochastic) and writes
   `phase3_inference/results/<model>/risk_radiorag.jsonl`.
4. Tears the vLLM server down.

For a single-model smoke test:

```bash
sbatch phase3_inference/slurm/run_debug.sh
```

> **Not regenerated by this release:** proprietary API entries
> (OpenAI / Gemini), very-large open-weight models we served from a
> separate vLLM endpoint (DeepSeek-R1, DeepSeek-V3.2, Mistral-Large-3,
> Qwen3-VL-235B, Llama-4-Scout-17B-16E), and any MIMIC-CXR vision tasks.
> All of these can be reproduced by pointing `02_run_inference.py` at a
> matching OpenAI-compatible endpoint.

## Phase 4 — LLM-judge answer verification

Every Phase-3 record is re-parsed by a stronger judge
(`Mistral-Small-4-119B-2603`, served over vLLM on the same node) to
recover the answer from free-text completions and flag abstentions.

```bash
sbatch phase4_checking_results/slurm/run_check.sh

# After inspection, re-judge any model flagged as unreliable in
# rejudge_report.json:
sbatch phase4_checking_results/slurm/run_rejudge.sh
```

Input : `phase3_inference/results/<model>/risk_radiorag.jsonl`
Output: `phase4_checking_results/results/<model>/risk_radiorag_checked.jsonl`

## Phase 5 — Bootstrap statistical analysis

Each script is independent, reads from
`phase4_checking_results/results/`, and writes into
`phase5_bootstrapping/bootstrap_results/`.

```bash
SIF=$WORKSPACE/environment/inference.sif
EXEC="apptainer exec --bind $WORKSPACE:$WORKSPACE $SIF python3"

$EXEC phase5_bootstrapping/run_bootstrap.py            # accuracy + 95 % CIs
$EXEC phase5_bootstrapping/run_safety_rates.py         # high-risk / unsafe / contradiction
$EXEC phase5_bootstrapping/run_confidence.py           # self-consistency confidence
$EXEC phase5_bootstrapping/run_per_question_safety.py  # per-question safety aggregation
$EXEC phase5_bootstrapping/run_ensembles.py            # majority-vote ensembles
$EXEC phase5_bootstrapping/batch_bootstrap.py          # batched bootstrap helper
$EXEC phase5_bootstrapping/run_pvalues.py              # paired-bootstrap p-values
```

`pvalue.py` is an importable utility used by `run_pvalues.py`, not a
stand-alone entry point.

The paper's Tables 1–4 are direct restylings of the JSON / CSV outputs
under `phase5_bootstrapping/bootstrap_results/`; no additional table-filling
script is shipped.

## Runtime on our development cluster

| Phase | Scale                                              | Wall-clock           |
| ----- | -------------------------------------------------- | -------------------- |
| 2     | ~150 k Radiopaedia chunks                          | ~30 min on 1×A100    |
| 3     | ~35 open-weight models × 7 conditions × (1 + 20)   | ~1 GPU-week (multi-node) |
| 4     | Same coverage, single judge model                  | ~6 h on 1×8-H100 node|
| 5     | Bootstrap 1 000 resamples across all models        | ~20 min on CPU       |

Phase 5 is cheap enough to re-run ad libitum during paper revisions.

## Supplementary materials

Supplementary tables and figures in the paper are downstream presentation
artefacts derived from the Phase-5 outputs. They are **not** regenerated
by this release; reviewers can rebuild them from
`phase5_bootstrapping/bootstrap_results/*.json` using their preferred
plotting / table tooling.
