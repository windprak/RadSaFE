# SaFE-Scale: code release

Source code accompanying the paper *“Safety and accuracy follow different
scaling laws in clinical large language models”* (RadSaFE-200).

This repository contains the **end-to-end pipeline** used to produce results from the paper:

1. **Phase 2 — RAG context construction.** Chunk the Radiopaedia corpus,
   embed it with BGE-large, build a FAISS index, retrieve top-1/5/10 chunks
   per question, and emit `risk_radiorag_final.jsonl` with all context
   conditions packed in.
2. **Phase 3 — Inference.** Run open-weight models on a local vLLM server
   (8×H100 / 8×H200) over the seven prompt conditions. Greedy +
   20-sample stochastic decoding for self-consistency.
3. **Phase 4 — LLM-judge answer verification.** Re-parse every model output
   with a stronger judge model to recover answers from free-text responses
   and to mark abstentions.
4. **Phase 5 — Statistical analysis.** Bootstrap accuracy, safety rates
   (high-risk, unsafe, contradiction), confidence calibration, ensemble
   majority votes, and per-question / per-condition tables. Produces the
   numbers reported in Tables 1–4 of the paper.

> Supplementary tables and figures are not regenerated by this release;
> they are downstream presentation artefacts that can be rebuilt directly
> from the Phase-5 JSON / CSV outputs.

> **Dataset scope.** **Only the RadSaFE-200 (`risk_radiorag`) dataset is
> reported in the paper.** All released code is trimmed to that dataset only.

## Layout

```
code_release/
├── environment/             apptainer (Singularity) recipes
├── phase2_rag_context/      Radiopaedia → FAISS → context injection
├── phase3_inference/        local vLLM inference (open-weight models)
├── phase4_checking_results/ LLM-judge answer verification
├── phase5_bootstrapping/    statistical analysis → Tables 1–4
└── docs/
    ├── HPC.md               cluster / SLURM / Apptainer notes
    └── REPRODUCING.md       end-to-end reproduction recipe
```

Each phase has its own `README.md` with the local invocation details.

## Hardware and execution model

This pipeline was developed and run on a **shared HPC cluster** (NHR@FAU
helma) using:

- **Apptainer / Singularity** for the runtime (`environment/*.def`).
- **SLURM** for batch scheduling (`phase{3,4}/slurm/*.sh`).
- 8×H100 / 8×H200 nodes for most open-weight models; B200 and MI300X nodes
  for the largest models (DeepSeek-R1/V3.2, Llama-4-Scout-17B-16E,
  Mistral-Large-3-675B, Qwen3-VL-235B). See `docs/HPC.md` for a single-GPU
  fallback.

## Quick start

1. Set the workspace root once:

   ```bash
   export WORKSPACE=/abs/path/to/this/checkout
   export MODELS_DIR=/abs/path/to/local/HF/model/cache    # for vLLM models
   ```

2. Build the two apptainer images (one-time):

   ```bash
   apptainer build environment/rag.sif        environment/rag.def
   apptainer build environment/inference.sif  environment/inference.def
   ```

3. Run the phases sequentially. See `docs/REPRODUCING.md` for the exact
   command list per phase.

## Software versions

Pinned versions live in the apptainer recipes (`environment/*.def`) and in
`environment/inference_packages.lock.txt`. Key components: vLLM,
BGE-large-en-v1.5 embeddings, FAISS-cpu, NumPy.
